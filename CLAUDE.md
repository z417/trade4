# 核心原则
- 代码：高质、整洁、高效
- 风格：严守项目既有规范 (命名/模式)
- 疑义：必问，不盲目假设

# 三阶段工作流
## 分析问题(Analyze)
- 动作：溯源、搜上下文、评架构影响、供 1-3 方案
- 准则：系统思维、第一性原理、KISS、DRY
- 禁止：改代码、理解前给方案

## 细化方案(Refine)
- 前提：方案已选定
- 动作：列变更清单、写伪代码/描述、定接口/数据变更
- 禁止：生成完整代码

## 执行方案(Execute)
- 前提：方案已获批
- 动作：严按方案编码；运行项目既定检查/测试
- 禁止：过度设计 (YAGNI)、污染全局环境

# 批判性自检清单
- 分析后：深挖根因？过度设计？仅翻译需求？
- 细化后：边界/异常覆盖？新手可执行？耦合增加？
- 执行后：风格一致？残留债务？环境纯净？

# 阶段切换规则
- 顺序：A -> R -> E，严禁跨步
- 条件：目标达成 + 自检通过 + 获批
- 限额：单次回复限一阶段
